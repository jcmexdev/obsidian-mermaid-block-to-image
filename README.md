# Mermaid Block to Image - Obsidian Plugin

An Obsidian plugin that identifies Mermaid diagram code blocks and converts them into static PNG images using the free, public [Kroki API](https://kroki.io).

## Key Features

- **Source Code Preservation**: The original Mermaid source code is commented out with native Obsidian comments (`%%`) directly above the generated image link.
- **Auto-Cleanup**: Re-converting a modified commented block automatically updates the image and removes the previous PNG file from your vault (trashing it safely) to avoid cluttering.
- **Configurable Storage**: Choose to save PNGs in the same folder as the current note (default) or in a custom vault folder.
- **Self-Hosting Friendly**: Customize the Kroki server URL in the settings to point to your own local/private instance.

## Installation

### Manual Installation
1. Create a directory named `obsidian-mermaid-block-to-image` under `<your-vault>/.obsidian/plugins/`.
2. Copy `main.js` and `manifest.json` into that directory.
3. Reload Obsidian, open settings, go to **Community plugins**, and enable **Mermaid Block to Image**.

## Usage

1. Open a note containing a Mermaid code block:
   ````markdown
   ```mermaid
   graph TD
       A --> B
   ```
   ````
2. Right-click inside the block and select **Convert Mermaid to PNG**, or run the **Convert Mermaid block to PNG** command from the command palette.
3. The plugin will:
   - Request the PNG from Kroki.
   - Save the image in your vault.
   - Comment out the source code block with a warning.
   - Insert the image link below the commented-out block:
     ```markdown
     %% [Autogenerated by Mermaid Block to Image. Do not delete or modify this block]
     ```mermaid
     graph TD
         A --> B
     ```
     %%
     ![[mermaid-c1a5b8e9.png]]
     ```

## Settings

- **Kroki instance URL**: Define the Kroki server address (default: `https://kroki.io`).
- **Storage location**:
  - `Same folder as the note`: Stores the PNG file in the same directory as the note.
  - `Custom folder in the vault`: Saves the PNG file in a specific path inside your vault.
- **Custom folder path**: Visible when using custom storage folder (e.g. `assets/diagrams`).
