# Mermaid Block to Image - Obsidian Plugin

![mermaid_banner.png](https://i.ibb.co/gLb2rKVG/mermaid-banner-png.png)

Convert Mermaid code blocks to static, local images (SVG, PNG, or WebP) completely offline and locally on your device. This speeds up note loading, preserves theme colors, and ensures consistent rendering across devices without using any external internet services.

> ⚠️ This plugin is designed to run in editing views (**Source Mode** and **Live Preview**). It does not perform conversion or show right-click menu items while in **Reading Mode**.

---

## Key Features

- ⚡ **Instant Conversion**: Convert diagrams directly via interactive hover buttons, the right-click context menu, or the Command Palette.
- 🔌 **100% Local & Offline**: All rendering is done directly on your device using Obsidian's built-in Mermaid engine. No internet connection or third-party servers required.
- 🔒 **Safe & Non-Destructive**: The original Mermaid source code is commented out with native Obsidian comments (`%%`) directly above the generated image. You never lose your code!
- 🔁 **Bidirectional Restore**: Convert any image block back into an active code block to edit it with live visual feedback. The plugin comments out the image link so you don't lose the file while editing.
- ♻️ **Smart Cleanup**: When you convert a restored block back to an image, the plugin automatically finds the previous commented image link, deletes the old file from your vault (moving it to the trash), and inserts the new image.
- 🖱️ **Interactive Hover Buttons**: Easily convert or restore blocks using the hover buttons (**Convert to [Format]** or **Restore code**) that appear in the top-right corner of diagrams and images.
- 📁 **Configurable Storage**: Choose to save images in the same folder as the active note (default) or in a custom folder of your choice (e.g., `assets/diagrams`).

---

## Usage

### Converting a Mermaid Block
1. Hover over any active Mermaid block and click the **Convert to [Format]** button in the top-right corner.
2. Alternatively, right-click inside the block and select **Convert to local [Format] image** (reflects your selected format: SVG, PNG, or WebP).
3. Or place your cursor inside the block, open the Command Palette (`Ctrl/Cmd + P`), and run **Convert Mermaid block to local image**.

### Restoring to a Code Block
1. Hover over any generated diagram image and click the **Restore code** button in the top-right corner.
2. Alternatively, right-click inside the commented block and select **Restore to Mermaid code block**.
3. Or place your cursor inside the commented block, open the Command Palette (`Ctrl/Cmd + P`), and run **Restore local image to Mermaid block**.

### Regenerating an Image
- If you edit the Mermaid code directly inside a commented block, right-click and select **Regenerate local [Format] image** to update it.

## How It Works

### 1. Active Diagram
You start with a standard Mermaid code block:

````markdown
```mermaid
graph TD
    A --> B
```
````

### 2. Convert to Image
Right-click inside the block and select **Convert to local SVG image**. The block is replaced by the commented source code and the active image link:
````markdown
%% [Autogenerated by Mermaid Block to Image Plugin. Do not delete or modify this line]
```mermaid
graph TD
    A --> B
```
[Autogenerated by Mermaid Block to Image Plugin. Do not delete or modify this line] %%
![[mermaid-c1a5b8e9.svg]]
````

### 3. Restore to Edit
If you want to edit the diagram with live visual feedback, right-click the block and select **Restore to Mermaid code block**. The code is restored and the image link is commented out so the file is not lost:
````markdown
```mermaid
graph TD
    A --> B
```
%% ![[mermaid-c1a5b8e9.svg]] %%
````

### 4. Regenerate & Clean Up
Once you finish editing, right-click and select **Convert to local SVG image** again. The plugin renders the new image, deletes the old `mermaid-c1a5b8e9.svg` file from your vault, and inserts the new image link!

## Installation

1. Open Obsidian and go to **Settings** ➔ **Community plugins**.
2. Click **Browse** and search for **Mermaid Block to Image**.
3. Click **Install**, then click **Enable**.

---

## Settings & Configuration

To customize the plugin, go to **Settings** ➔ **Mermaid Block to Image**:

- **Local Image Format**: Choose between SVG, PNG, or WebP. SVG is recommended for crisp vector diagrams that dynamically adapt to dark/light theme changes.
- **Storage Location**:
  - `Same folder as the note`: Saves the generated image files right next to the markdown note.
  - `Custom folder in the vault`: Saves all images in a central directory of your choice.
- **Custom Folder Path**: The directory inside your vault where images will be saved if using a custom location (e.g., `resources/attachments`).

## FAQ

<details>
<summary><b>Why choose SVG format?</b></summary>
<br>

SVG is a vector format. It remains perfectly crisp at any zoom level, has a very small file size, and preserves the theme colors of your Obsidian vault. It is the recommended format.
</details>

<details>
<summary><b>Does this plugin send my diagrams to the internet?</b></summary>
<br>

No. All conversion is done 100% locally on your computer or mobile device. No internet connection is used or required.
</details>

<details>
<summary><b>How do I edit a diagram that has already been converted?</b></summary>
<br>

You have two options:
1. **Directly edit the comments**: Modify the Mermaid code directly inside the commented `%%` block in your note. Once edited, right-click inside the block and select **Regenerate local [Format] image**.
2. **Restore to code block**: Right-click the block and select **Restore to Mermaid code block**. This brings back the active code block for live preview editing and comments out the image link. Once you are done, convert it back to an image. The plugin will automatically clean up the old file for you.
</details>

<details>
<summary><b>What happens if I manually delete the generated image from my vault?</b></summary>
<br>

The plugin will handle it gracefully. It will skip any file deletion operations and simply let you re-convert or regenerate the diagram, creating a new image file without throwing errors.
</details>
